# Support

## Before opening a ticket

1. Check the [FAQ](faq.md).
2. Run through [Troubleshooting](troubleshooting.md).
3. Confirm the same flow works on the website. If it fails on the website, the fix belongs there, not in the app.

## Channels

| Channel | URL | Best for |
|---|---|---|
| Documentation | https://docs.botble.com/ecommerce-mobile-app | Setup, configuration |
| Support tickets | https://botble.ticksy.com | Bugs, licensing, technical issues |
| Email | contact@botble.com | Business / partnership |

Typical response times (business days):

| Priority | Response |
|---|---|
| Critical (app not working) | 24–48 hours |
| High (major feature broken) | 2–3 days |
| Normal | 3–5 days |
| Feature request | 1–2 weeks |

## What support covers

- Installation, setup, configuration issues
- Bugs in the unmodified source
- Documentation clarification
- Basic customization questions

Not covered:

- Custom feature development
- Third-party integrations
- Hosting / server troubleshooting
- Issues caused by modified source

For paid customization or setup work, email contact@botble.com.

## What to include in a ticket

- Purchase code
- App version, Expo SDK, Node.js version, OS
- Exact error (full text or screenshot)
- Steps to reproduce
- What you have already tried

Do not send passwords, API keys, customer data, or your full Envato purchase code via insecure channels.
